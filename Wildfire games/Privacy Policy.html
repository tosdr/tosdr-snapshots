5The 0 A.D. UserReporter is a tool that allows you, the 0 A.D. player, to automatically upload hardware- and software-system details in order to help Wildfire Games and community developers to improve the performance and compatibility of 0 A.D.
6
7[font="sans-bold-14"]Purpose of the UserReport:[/font]
8The UserReports indicate which system features (for example CPU instruction sets and OpenGL capabilities) are widely supported by the computer systems of 0 A.D. players.
9This allows developers to decide which 0 A.D. program code is safe to use and which optimizations are most feasible to implement next.
10If the UserReporter is enabled, the UserReport containing that information is uploaded to Wildfire Games once per program launch (GDPR 13.1.c, GDPR 15.1.a).
11
12[font="sans-bold-14"]UserReport data:[/font]
13 1. Hardware Details, for example the screen size, CPU clock rate, memory capacity and the sound card manufacturer.
14 2. System Software Details, for example the operating system version, graphics driver version, OpenGL capabilities.
15 3. Application Details, for example the 0 A.D. version and build number.
16 4. The UserID. This is a pseudonym generated by the UserReporter the first time it is used (GDPR 4.5, GDPR 25.1, GDPR 32.1.a).
17 The UserID enables developers to count how many 0 A.D. players use a specific set of hardware or system software without allowing them to attribute the data to any person.
18 5. The date of the upload of the UserReport. This allows focus on recent UserReports and disregard old reports.
19
20The UserReport does not include any specially protected categories of personal data (GDPR 9, GDPR 10).
21The transmission is secured with SSL. Data is protected against unintentional loss in encrypted backups for additional time (GDPR 30.1.g, GDPR 32).
22

25 At the time of the document, Wildfire Games published the most recent UserReport analysis at http://feedback.wildfiregames.com/ (GDPR 13.1.e, GDPR 15.1.c).
26 2. Erase any UserReports for any reason, except where a user has objected to the erasure of his or her data for one of the reasons specified by the GDPR (GDPR 18, GDPR 21).
27 3. Store the users IP address to protect against cyberattacks, for no longer than 4 months (EU Court of Justice Press Release No 112/16).
28 4. Change this document. The document date indicates the version of the Terms and Conditions.
29 The UserReport is disabled with each version update and only enabled if the user agreed to the new document (GDPR 13.3).
